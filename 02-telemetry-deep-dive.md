# Windows 遥测深度解析：你的数据包到底装了什么？

## 系列文章

这是 Windows 隐私系列的第二章。第一章「[你的数据去哪了？](https://github.com/3612hdu/windows-privacy-guide)」涵盖了六类隐私设置的概览。

本章聚焦遥测系统——这是 Windows 隐私体系中最复杂、数据量最大的部分。

---

## 一、遥测不是什么

很多人把 Windows 遥测和"微软在监视你"划等号。实际情况更微妙：

**遥测不是：**
- 微软员工在后台看你的屏幕（技术上不可行，数据量太大）
- 你的个人文件被上传（Word 文档、照片等不会被作为遥测数据发送）
- 键盘记录器（不记录具体的按键序列）

**遥测是：**
- 系统级事件的自动上报：应用崩溃、驱动错误、更新失败
- 使用统计：哪些功能被用了、用了多久、在哪一步放弃了
- 设备状态：CPU/内存/磁盘的使用模式、电池消耗
- **在"完整"级别下**：访问过的网站 URL、应用使用详情、搜索词

## 二、遥测数据的具体分类

根据微软的公开文档，诊断数据分为以下类别：

### 2.1 通用数据（所有级别都收集）

| 类别 | 示例 | 用途 |
|------|------|------|
| 设备信息 | 制造商、型号、CPU 类型、内存大小 | 判断哪些硬件能运行 Windows |
| 系统信息 | OS 版本、语言、时区、更新状态 | 了解用户群体分布 |
| 质量数据 | 应用崩溃堆栈、驱动错误码、蓝屏信息 | 修复 bug |
| 安装数据 | 应用安装/卸载、更新成功/失败率 | 改善更新体验 |

### 2.2 增强级别额外收集

| 类别 | 示例 |
|------|------|
| 应用使用 | 哪些应用被打开、使用频率、使用时长 |
| 联网状况 | 网络类型（Wi-Fi/以太网）、连接质量、数据使用量 |
| 系统配置 | 打印机设置、显示配置、电源方案 |

### 2.3 完整级别额外收集

| 类别 | 示例 |
|------|------|
| 浏览历史 | 通过 Edge 和 WebView 访问的 URL（不是其他浏览器） |
| 搜索词 | 在开始菜单和 Windows 搜索中输入的查询 |
| 应用内行为 | 在 UWP 应用中点击了哪些按钮、停留了多久 |
| 墨迹/语音样本 | 手写输入、语音命令的采样数据 |

## 三、你在完整模式下会发什么？（真实案例）

以下是一个典型的"完整(3)"级别遥测上报场景：

**场景：你用 Edge 浏览器打开了一个购物网站，搜索了"蓝牙耳机"，然后在搜索结果中点击了第三个链接。**

完整遥测会上报：
1. Edge 浏览器被激活（应用启动事件）
2. 导航到 `https://www.example-shopping.com`（URL 记录）
3. 在搜索框中输入"蓝牙耳机"（搜索词记录——**仅限 Edge 和 Windows 搜索，不包括 Chrome/Firefox**）
4. 点击了搜索结果第 3 项（用户交互事件）
5. 页面加载耗时 2.3 秒（性能指标）

**基本(1)遥测只上报：**
1. Edge 浏览器被激活（无 URL）
2. 没有崩溃事件

差别巨大。

## 四、诊断数据查看器——你自己验证

Microsoft 在商店中提供了一个官方工具：**诊断数据查看器**（Diagnostic Data Viewer）。

安装后，你可以：
- 看到你的设备实际上报了哪些诊断事件
- 按类别筛选（浏览历史、应用使用、设备连接等）
- 搜索特定应用或时间段的报告

怎么用：
1. 打开设置 → 隐私 → 诊断和反馈
2. 打开"诊断数据查看器"开关
3. 点击"诊断数据查看器"按钮，跳转到 Microsoft Store 安装
4. 打开后，你会看到一个按时间排列的事件列表

**这才是 PrivacyScope 这类工具做不到的事——让你亲眼看到数据包里装了什么。**

## 五、遥测的网络流量特征

Windows 遥测连接到以下 Microsoft 端点：

| 端点 | 用途 |
|------|------|
| `vortex.data.microsoft.com` | 主要遥测上传端点 |
| `settings-win.data.microsoft.com` | 设置同步 |
| `watson.telemetry.microsoft.com` | 崩溃/错误报告 |
| `oca.telemetry.microsoft.com` | Office 遥测 |
| `v10.events.data.microsoft.com` | 事件追踪 |

如果你在路由器或防火墙层面阻止了这些域名，Windows 遥测就无法上报。但：
- Windows Update 可能也会受影响（共享部分基础设施）
- 一些 Microsoft 服务（如商店、OneDrive）可能异常

## 六、企业版的特殊待遇

Windows 企业版和教育版可以将遥测级别设为 **0（仅安全）**：

| 级别 | 家庭版 | 专业版 | 企业版/教育版 |
|------|--------|--------|-------------|
| 0 - 仅安全 | ❌ | ❌ | ✅ |
| 1 - 基本 | ✅ | ✅ | ✅ |
| 2 - 增强 | ✅ | ✅ | ✅ |
| 3 - 完整 | ✅（默认） | ✅（默认） | ✅ |

这是一个重要的政策差异：**个人用户和小企业无法完全关闭遥测。** 微软的解释是"我们需要足够的数据来维持 Windows 的质量"。

专业版用户可以通过组策略将遥测设为 0，但微软明确表示这不被支持，且可能在未来更新中被忽略。

---

## 七、多少人真的关掉了？

微软没有公开发布遥测级别的用户分布数据。但从第三方研究推断：
- 约 **70-80%** 的用户保持默认的"完整(3)"级别（因为不知道这个设置存在）
- 约 **10-15%** 设为"基本(1)"（通常是企业 IT 部门统一配置）
- 仅 **5%** 以下使用企业版的"仅安全(0)"

这意味着绝大多数 Windows 用户的浏览历史和搜索词在持续上报。

---

## 八、你应该关掉完整遥测吗？

**这是一个需要你自己做的判断。** 以下是非导向性的事实对比：

| 保持完整(3) | 切换到基本(1) |
|------------|-------------|
| Microsoft 能更快发现和修复你遇到的 bug | 崩溃报告仍然会上报（但不含上下文） |
| Windows 更新和驱动推荐更精准 | 驱动更新可能不够精准 |
| 你的浏览历史和搜索词被记录 | 浏览历史和搜索词不上报 |
| Microsoft 用你的数据训练 AI 模型（已公告） | 你的数据不参与 AI 训练 |

**两件事同时成立：**
1. 微软确实用遥测数据改善了 Windows 的稳定性和用户体验
2. 微软也确实用这些数据做了一些用户不知情的事（如 AI 训练）
